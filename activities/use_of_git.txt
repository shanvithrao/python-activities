==============================================
         THE USE OF GIT IN DEVELOPMENT
==============================================

Git is one of the most essential tools in modern software development.
Below are the key uses and benefits of Git:

1. VERSION CONTROL
-------------------
Git keeps a complete history of every change made to a project. You can
go back to any previous version of your code at any time. This prevents
data loss and allows you to undo mistakes easily.

2. COLLABORATION
-----------------
Multiple developers can work on the same project simultaneously. Git
manages changes from different team members and helps merge them together
without conflicts (or helps resolve conflicts when they occur).

3. BRANCHING AND EXPERIMENTATION
----------------------------------
Developers can create separate branches to work on new features or bug fixes
without affecting the main (stable) codebase. Once the work is tested and
reviewed, it can be merged back into the main branch.

Example Workflow:
  main branch        -> stable, production-ready code
  feature/login      -> new login feature being developed
  bugfix/fix-crash   -> fixing a reported bug

4. BACKUP AND RECOVERY
-----------------------
By pushing code to a remote repository (like GitHub), teams always have a
backup of their code. If a local machine crashes, the code is safe in the cloud.

5. CODE REVIEW
---------------
Using platforms like GitHub, team members can review each other's code via
Pull Requests before merging changes. This improves code quality and reduces bugs.

6. CONTINUOUS INTEGRATION / CONTINUOUS DEPLOYMENT (CI/CD)
-----------------------------------------------------------
Git integrates with CI/CD tools (like GitHub Actions, Jenkins, CircleCI) to
automatically test and deploy code whenever changes are pushed. This speeds
up the development lifecycle.

7. OPEN SOURCE CONTRIBUTION
-----------------------------
Git enables developers worldwide to contribute to open-source projects by
forking repositories, making changes, and submitting pull requests to the
original project.

8. AUDIT TRAIL
---------------
Every commit in Git records:
- What changed
- Who made the change
- When the change was made
- Why the change was made (via commit messages)

This creates a complete audit trail useful for debugging and accountability.

COMMON GIT COMMANDS:
---------------------
  git init           - Initialize a new Git repository
  git clone <url>    - Clone an existing repository
  git status         - Check the status of your working directory
  git add .          - Stage all changes for commit
  git commit -m ""   - Commit staged changes with a message
  git push           - Push commits to a remote repository
  git pull           - Pull latest changes from remote
  git branch         - List, create, or delete branches
  git checkout       - Switch branches or restore files
  git merge          - Merge branches together
  git log            - View commit history
